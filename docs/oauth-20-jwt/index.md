# OAuth 2.0 & JWT

## Introduction to OAuth 2.0
OAuth 2.0 is an authorization framework that enables applications to obtain limited access to user resources on another service provider's website, without sharing their login credentials. It's a widely adopted standard, used by companies like Google, Facebook, and GitHub. In this section, we'll delve into the basics of OAuth 2.0, its components, and how it works.

The OAuth 2.0 flow involves the following components:
* **Resource Server**: The server that protects the resources the client wants to access.
* **Authorization Server**: The server that authenticates the user and issues an access token.
* **Client**: The application that requests access to the protected resources.
* **Access Token**: A token that the client uses to access the protected resources.

Here's an example of how the OAuth 2.0 flow works:
1. The client requests authorization from the authorization server.
2. The authorization server redirects the user to a login page.
3. The user logs in and grants access to the client.
4. The authorization server redirects the user back to the client with an authorization code.
5. The client exchanges the authorization code for an access token.
6. The client uses the access token to access the protected resources.

### OAuth 2.0 Grant Types
OAuth 2.0 supports several grant types, each with its own use case:
* **Authorization Code Grant**: Used for server-side applications that need to access protected resources.
* **Implicit Grant**: Used for client-side applications that need to access protected resources.
* **Resource Owner Password Credentials Grant**: Used for applications that need to access protected resources using the user's login credentials.
* **Client Credentials Grant**: Used for applications that need to access protected resources without user interaction.

## Introduction to JWT
JSON Web Tokens (JWT) is a compact, URL-safe means of representing claims to be transferred between two parties. JWT is digitally signed and contains a payload that can be verified and trusted. In this section, we'll explore the basics of JWT, its components, and how it works.

A JWT token consists of three parts:
* **Header**: The header contains the algorithm used for signing the token.
* **Payload**: The payload contains the claims or data that the token asserts.
* **Signature**: The signature is generated by signing the header and payload with a secret key.

Here's an example of a JWT token:
```json
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "1234567890",
    "name": "John Doe",
    "admin": true
  },
  "signature": "HMACSHA256(
    base64UrlEncode(header) + "." +
    base64UrlEncode(payload),
    secret_key
  )"
}
```
### JWT Use Cases
JWT is commonly used for authentication and authorization in web applications. Here are some examples of JWT use cases:
* **Authentication**: JWT can be used to authenticate users by verifying the token's signature and payload.
* **Authorization**: JWT can be used to authorize users by verifying the token's payload and checking for specific claims.
* **Single Sign-On (SSO)**: JWT can be used to implement SSO by generating a token that can be used to access multiple applications.

## Implementing OAuth 2.0 with JWT
In this section, we'll explore how to implement OAuth 2.0 with JWT using a real-world example. We'll use the following tools and platforms:
* **Node.js**: A JavaScript runtime environment for building server-side applications.
* **Express.js**: A Node.js web framework for building web applications.
* **Passport.js**: A Node.js library for authentication.
* **jsonwebtoken**: A Node.js library for generating and verifying JWT tokens.

Here's an example of how to implement OAuth 2.0 with JWT using Node.js and Express.js:
```javascript
const express = require('express');
const passport = require('passport');
const jwt = require('jsonwebtoken');

const app = express();

// Set up passport.js
passport.use(new LocalStrategy(
  (username, password, done) => {
    // Verify the user's credentials
    if (username === 'john' && password === 'hello') {
      return done(null, { id: 1, username: 'john' });
    }
    return done(null, false);
  }
));

// Set up JWT strategy
passport.use(new JWTStrategy(
  {
    jwtFromRequest: (req) => {
      let token = null;
      if (req && req.cookies) {
        token = req.cookies['jwt'];
      }
      return token;
    },
    secretOrKey: 'secret_key',
  },
  (payload, done) => {
    // Verify the token's payload
    if (payload && payload.id) {
      return done(null, { id: payload.id, username: payload.username });
    }
    return done(null, false);
  }
));

// Generate a JWT token
app.post('/login', (req, res) => {
  const user = { id: 1, username: 'john' };
  const token = jwt.sign(user, 'secret_key', { expiresIn: '1h' });
  res.cookie('jwt', token);
  res.send({ token });
});

// Protect a route with JWT authentication
app.get('/protected', passport.authenticate('jwt', { session: false }), (req, res) => {
  res.send({ message: 'Hello, ' + req.user.username });
});
```

## Common Problems and Solutions
In this section, we'll explore some common problems and solutions when implementing OAuth 2.0 with JWT:
* **Token Expiration**: The token expires after a certain period of time.
	+ Solution: Use a refresh token to obtain a new access token.
* **Token Revocation**: The token is revoked or invalidated.
	+ Solution: Use a token blacklist to verify the token's validity.
* **CSRF Attacks**: An attacker tricks the user into performing an unintended action.
	+ Solution: Use a CSRF token to verify the request's origin.

Here are some best practices for implementing OAuth 2.0 with JWT:
* **Use HTTPS**: Use HTTPS to encrypt the communication between the client and server.
* **Use a Secure Secret Key**: Use a secure secret key to sign and verify the JWT token.
* **Use a Token Blacklist**: Use a token blacklist to verify the token's validity.
